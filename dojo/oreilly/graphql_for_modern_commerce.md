# 1. Introducing GraphQL
## Commerce Requires More Than REST
**REST APIの課題**

REST APIは優れていますが、すべての変換テクノロジーと同様に、新たな課題がなければ利益は得られません。
- 仕様の欠如
- 呼び出すAPIがわからない
- データのオーバーフェッチ
- データのアンダーフェッチ

## What Is GraphQL?
```
query {
	product(id: "94695736", locale: "en_US") {
		brandIconURL
		name
		description
		price
		ableToSell
		averageReview
		numberOfReviews
		maxAllowableQty
		images {
			url
			altText
		}
	}
}
```

**GraphQLの利点**
- より簡単なフロントエンド開発
- パフォーマンスの向上
- 維持するコードが少ない

**GraphQLの欠点**
- セキュリティ
- 複数のGraphQLエンドポイントとスキーマを組み合わせることが難しい
